Blueprint Format Instructions

Each section defines a type of problem to include, either math or date, followed by how many of those problems to generate. 

<category>: <count>
    <option 1>
    <option 2>
    ...
  • <category>: Either math or date
  • <count>: Number of problems to generate in this block

Math Category Options

  • int [start] end — Random integers (default start = 0)
  • float [start] end — Random floats (default start = 0.0)
  • float.precision [start] end — Set precision (e.g., float.2)
  • op + - * / // % ** — Allowed operators
  • func — Valid functions:
    • abs, ceil, floor, round, exp, log, log10, sqrt, sin, cos, tan
  • const — Valid constants:
    • c, h, hbar, G, e, k, N_A, R, alpha, mu_0, epsilon_0, sigma, zero_Celsius, pi, Avogadro, Boltzmann, Planck, speed_of_light, elementary_charge, gravitational_constant
    • You can also set your own integer or float values as constants (e.g., const 42 or const 3.14).
  • ( or ) — Use brackets

Date Category Options

  • start <year> — Optional, default 1900
  • end <year> — Optional, default 2050

Memory Category Options

  • <key> <value> [on|off] — Optional, default = on

Note: Indent lines inside blocks. Use spaces or tabs. Blank lines are allowed. Errors in structure will trigger configuration errors.
